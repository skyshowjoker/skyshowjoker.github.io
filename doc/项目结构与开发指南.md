# 项目结构与开发指南

## 项目概述

这是一个基于 **AstroPaper v5.5.1** 的现代化博客网站，部署在 GitHub Pages 上。

- **网站地址**: https://skyshowjoker.github.io
- **作者**: Perlicue
- **框架**: Astro v5.16.6
- **包管理器**: pnpm v10.11.1

---

## 核心技术栈

### 前端框架
- **Astro** v5.16.6 - 静态站点生成器，支持部分水合
- **TypeScript** v5.9.3 - 类型安全

### 样式系统
- **TailwindCSS** v4.1.18 - 实用优先的 CSS 框架
- **@tailwindcss/typography** v0.5.19 - 排版插件

### 内容管理
- **remark-toc** v9.0.0 - 目录生成
- **remark-collapse** v0.1.2 - 可折叠区块
- **@shikijs/transformers** v3.20.0 - 代码语法高亮

### 图片处理
- **@resvg/resvg-js** v2.6.2 - SVG 渲染
- **satori** v0.18.3 - 动态 OG 图片生成
- **sharp** v0.34.5 - 图片处理

### SEO 与搜索
- **@astrojs/sitemap** v3.6.0 - 站点地图生成
- **@astrojs/rss** v4.0.14 - RSS 订阅源
- **pagefind** v1.4.0 - 静态搜索索引

### 开发工具
- **ESLint** v9.39.2 - 代码检查
- **Prettier** v3.7.4 - 代码格式化
- **Docker** - 容器化支持

## 主要功能

### 核心特性
✅ **类型安全的 Markdown** - 使用 Zod Schema 验证博客文章
✅ **超快性能** - 静态生成，最小化 JavaScript
✅ **无障碍访问** - 语义化 HTML、ARIA 标签、键盘导航
✅ **响应式设计** - 移动端优先的 TailwindCSS 方案
✅ **SEO 友好** - 站点地图、RSS 订阅、OG 图片生成

### 用户体验
✅ **明暗主题切换** - 通过 `theme.ts` 脚本实现
✅ **模糊搜索** - Pagefind 静态搜索集成
✅ **草稿文章** - 支持草稿和定时发布
✅ **分页功能** - 可配置每页文章数量
✅ **动态 OG 图片** - 为每篇博客文章生成

### 内容组织
✅ **标签系统** - 文章按标签组织，带专用页面
✅ **归档页面** - 可选的旧文章归档页面
✅ **社交分享** - 内置分享链接（WhatsApp、Facebook、X、Telegram、Pinterest、Email）
✅ **编辑链接** - 可选的 "编辑页面" 链接到 GitHub

## 配置指南

### 1. 站点基础配置

编辑 `src/config.ts`:

```typescript
export const SITE = {
  website: "https://skyshowjoker.github.io", // 你的网站 URL
  author: "你的名字",                         // 作者名称
  title: "你的博客标题",                      // 网站标题
  description: "博客描述",                    // 网站描述
  timezone: "Asia/Shanghai",                  // 时区
  postPerPage: 4,                             // 每页文章数量
  scheduledPostMargin: 15 * 60 * 1000,       // 定时发布边距（毫秒）
};

export const LOCALE = {
  lang: "zh-CN",      // HTML lang 属性
  langTag: ["zh-CN"], // BCP 47 语言标签
};

export const LOGO_IMAGE = {
  enable: false,      // 是否启用 Logo 图片
  svg: true,          // 是否使用 SVG
  width: 216,         // 宽度
  height: 46,         // 高度
};

export const FEATURES = {
  lightAndDarkMode: true,  // 明暗主题切换
  archives: true,          // 归档页面
  backButton: true,        // 返回按钮
  dynamicOgImages: true,   // 动态 OG 图片
  editLink: {
    enable: true,
    url: "https://github.com/skyshowjoker/skyshowjoker.github.io/edit/main/src/data/blog",
  },
};
```

## 创建博客文章

### 文章位置

在 `src/data/blog/` 目录下创建 `.md` 文件。支持子目录组织。

### Frontmatter 格式

```markdown
---
title: "文章标题"
description: "文章描述，用于 SEO"
pubDatetime: 2026-04-16T10:00:00+08:00
modDatetime: 2026-04-16T12:00:00+08:00
author: "作者名称"
tags: ["标签1", "标签2", "标签3"]
draft: false
featured: true
canonicalURL: "https://example.com/original-post"
---

## 文章内容

这里是文章的正文内容...

### 子标题

更多内容...
```

### Frontmatter 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | ✅ | 文章标题 |
| `description` | string | ❌ | 文章描述（SEO） |
| `pubDatetime` | ISO 8601 | ✅ | 发布时间 |
| `modDatetime` | ISO 8601 | ❌ | 修改时间 |
| `author` | string | ❌ | 作者名称（默认使用 config.ts 中的） |
| `tags` | string[] | ❌ | 标签数组 |
| `draft` | boolean | ❌ | 是否为草稿（默认 false） |
| `featured` | boolean | ❌ | 是否为精选文章（默认 false） |
| `canonicalURL` | string | ❌ | 规范链接（用于转载文章） |

### 时间格式示例

```typescript
// ISO 8601 格式
pubDatetime: 2026-04-16T10:00:00Z           // UTC 时间
pubDatetime: 2026-04-16T10:00:00+08:00      // 东八区时间
pubDatetime: 2026-04-16T10:00:00.000Z       // 带毫秒
```

### 草稿与定时发布

```markdown
---
title: "草稿文章"
draft: true  # 不会在生产环境显示
---
```

```markdown
---
title: "定时发布文章"
pubDatetime: 2026-12-31T00:00:00+08:00  # 未来时间，暂不显示
---
```

## 添加新页面

### 创建静态页面

在 `src/pages/` 创建 `.astro` 文件：

```astro
---
// src/pages/projects.astro
import Layout from "@layouts/Layout.astro";
import Header from "@components/Header.astro";
import Footer from "@components/Footer.astro";

const title = "我的项目";
---

<Layout title={title}>
  <Header activeNav="projects" />
  <main id="main-content">
    <section class="mx-auto max-w-3xl px-4 py-8">
      <h1 class="text-3xl font-bold">项目列表</h1>
      <!-- 你的内容 -->
    </section>
  </main>
  <Footer />
</Layout>
```

### 创建 Markdown 页面

在 `src/pages/` 创建 `.md` 文件：

```markdown
---
layout: ../layouts/AboutLayout.astro
title: "关于我"
---

## 个人简介

这里是关于我的介绍...
```

### 更新导航菜单

编辑 `src/components/Header.astro`，添加新的导航链接。

## 部署到 GitHub Pages

### 自动部署

项目已配置 GitHub Pages，推送到 `main` 分支即可自动部署：

```bash
git add .
git commit -m "更新内容"

```

`.github/workflows/ci.yml` 会自动：
1. 检查代码（ESLint）
2. 检查格式（Prettier）
3. 构建项目（Astro）

### 手动部署

```bash
# 构建项目
pnpm build

# 输出目录：dist/
# 将 dist/ 内容部署到 GitHub Pages
```

## 故障排除

### 构建失败

```bash
# 清理缓存
rm -rf node_modules .astro dist
pnpm install
pnpm build
```

### 搜索不工作

确保构建时包含 Pagefind：

```bash
pnpm build
# 会自动运行：pagefind --site dist
```

### 样式不生效

检查 TailwindCSS 配置，确保扫描了正确的文件路径。

### 时区问题

在 `src/config.ts` 中设置正确的时区：

```typescript
export const SITE = {
  timezone: "Asia/Shanghai", // 或其他时区
};
```
