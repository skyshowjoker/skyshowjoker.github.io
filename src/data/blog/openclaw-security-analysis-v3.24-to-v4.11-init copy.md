# OpenClaw 安全分析报告：v3.24 到 v4.11

## 概述

本报告分析了 OpenClaw 从版本 3.24 到 4.11 期间的重要安全变更和修复。OpenClaw 是一个 AI 代理框架，其安全模型基于"受信任的操作员"（Trusted Operator）模型，强调单一用户、单一网关的设计理念。

## 版本 3.28

### 安全案例

**问题标题**: 基础 SSRF 防护框架
**漏洞描述**: 早期版本缺乏系统性的 SSRF 防护机制。
**解决方案**:
- 引入 DNS 固定的调度器
- 实施私有 IP 地址检查
- 建立 URL 验证框架
- **版本**: 2026.3.28

**问题标题**: 媒体服务器路径遍历漏洞
**漏洞描述**: 媒体服务器未正确验证文件路径，可能允许访问预期目录之外的文件。
**解决方案**:
- 实施严格的路径验证
- 阻止路径遍历攻击
- **Git Commit**: b94b220156

**问题标题**: iMessage SCP 远程路径 shell 元字符注入
**漏洞描述**: iMessage 的 SCP 远程路径未正确清理，可能允许 shell 元字符注入。
**解决方案**:
- 清理 SCP 远程路径
- 防止 shell 元字符注入
- **Git Commit**: a54bf71b4c

**问题标题**: 未引用 heredoc 主体中的命令替换
**漏洞描述**: 未引用的 heredoc 主体可能执行命令替换。
**解决方案**:
- 阻止未引用 heredoc 主体中的命令替换
- **Git Commit**: 92cada2aca, a54bba6ee0

## 版本 3.31

### 高危安全案例

**问题标题**: Skills/Plugins 安装危险代码默认行为变更
**漏洞描述**: 之前版本中，危险代码检测和安装时扫描失败默认允许安装，攻击者可通过恶意插件安装后门代码。
**解决方案**:
- 内置危险代码 `critical` 发现和安装时扫描失败现在默认失败关闭
- 插件安装和网关支持的技能依赖安装现在可能需要显式的危险覆盖（如 `--dangerously-force-unsafe-install`）才能继续
- **版本**: 2026.3.31 (#57777, #57691)

**问题标题**: Gateway/auth trusted-proxy 认证绕过
**漏洞描述**: `trusted-proxy` 模式接受混合共享令牌配置，本地直连回退隐式认证同主机调用者。
**解决方案**:
- `trusted-proxy` 现在拒绝混合共享令牌配置
- 本地直连回退需要配置的令牌而不是隐式认证同主机调用者
- **版本**: 2026.3.31 Thanks @zhangning-agent, @jacobtomlinson

**问题标题**: Gateway/node commands 节点命令未授权暴露
**漏洞描述**: 设备配对足以暴露声明的节点命令，即使未经节点配对批准。
**解决方案**:
- 节点命令现在保持禁用状态直到节点配对被批准
- 设备配对本身不再足以暴露声明的节点命令
- **版本**: 2026.3.31 (#57777)

**问题标题**: Gateway/node events 节点事件信任边界
**漏洞描述**: 节点发起的运行依赖更广泛的主机/会话工具访问，可能超越预期权限边界。
**解决方案**:
- 节点发起的运行现在保持在减少的受信任表面上
- 依赖更广泛主机/会话工具访问的通知驱动或节点触发流可能需要调整
- **版本**: 2026.3.31 (#57691)

### 中危安全案例

**问题标题**: Nostr inbound DMs 签名验证缺失
**漏洞描述**: 入站 DM 事件在配对或发送者授权副作用之前未验证签名，伪造的 DM 事件可创建配对请求或触发回复尝试。
**解决方案**:
- 在配对或发送者授权副作用之前验证入站事件签名
- 伪造的 DM 事件不再创建配对请求或触发回复尝试
- **版本**: 2026.3.31 Thanks @smaeljaish771

**问题标题**: ACP/security 危险工具自动审批
**漏洞描述**: ACP 的危险工具名称覆盖允许非执行能力的工具自动审批，可能绕过安全检查。
**解决方案**:
- 用语义审批类替换 ACP 的危险工具名称覆盖
- 只有窄的只读读取/搜索可以自动审批
- 间接执行能力和控制平面工具始终需要显式提示审批
- **版本**: 2026.3.31

**问题标题**: Gateway/device tokens 活跃会话未断开
**漏洞描述**: 令牌轮换后活跃的设备会话继续连接，旧凭证仍然有效。
**解决方案**:
- 令牌轮换后断开活跃的设备会话
- 新轮换的凭证立即撤销现有的实时连接
- 不再等待这些套接字自然关闭
- **版本**: 2026.3.31 Thanks @zsxsoft

**问题标题**: Gateway/auth 浏览器 Origin 头验证
**漏洞描述**: 受信任代理 HTTP 操作员请求上未拒绝不匹配的浏览器 `Origin` 头。
**解决方案**:
- 在受信任代理 HTTP 操作员请求上拒绝不匹配的浏览器 `Origin` 头
- 保持无源的无头代理客户端正常工作
- **版本**: 2026.3.31 Thanks @AntAISecurityLab

**问题标题**: Gateway/auth 共享认证速率限制绕过
**漏洞描述**: 当调用者同时发送设备令牌候选时，共享认证速率限制被抑制，伪造的设备令牌字段可抑制共享密钥暴力跟踪。
**解决方案**:
- 在 WebSocket 握手尝试期间保持共享认证速率限制活动
- 即使调用者也发送设备令牌候选
- 伪造的设备令牌字段不再抑制共享密钥暴力跟踪
- **版本**: 2026.3.31 Thanks @kexinoh

**问题标题**: Exec/env 代理和 TLS 环境变量覆盖
**漏洞描述**: 请求作用域的命令可通过环境变量静默重定向出站流量或信任攻击者提供的证书设置。
**解决方案**:
- 在主机执行中阻止代理、TLS 和 Docker 端点环境变量覆盖
- 请求作用域的命令不能静默重路由出站流量或信任攻击者提供的证书设置
- **版本**: 2026.3.31 Thanks @AntAISecurityLab

**问题标题**: Host exec/env 编译器和包路径覆盖
**漏洞描述**: 请求作用域的环境变量覆盖可重定向 Docker 端点、信任根、编译器包含路径、包解析或 Python 环境根。
**解决方案**:
- 阻止额外的请求作用域环境变量覆盖
- 防止重定向 Docker 端点、信任根、编译器包含路径、包解析或 Python 环境根
- **版本**: 2026.3.31 Thanks @tdjackey

**问题标题**: Heartbeat/auth 非所有者工具访问继承
**漏洞描述**: 执行事件心跳运行从会话传递目标继承所有者专用工具访问，即使目标会话属于所有者。
**解决方案**:
- 防止执行事件心跳运行从会话传递目标继承所有者专用工具访问
- 节点执行输出保持在非所有者工具表面上
- **版本**: 2026.3.31 Thanks @AntAISecurityLab

**问题标题**: Feishu/groups 引用回复和主题引导上下文
**漏洞描述**: 引用回复和主题引导上下文未与组发送者允许列表对齐，非允许列表的线程消息可种子代理上下文。
**解决方案**:
- 将引用回复和主题引导上下文与组发送者允许列表对齐
- 只有允许列表的线程消息可种子代理上下文
- **版本**: 2026.3.31 Thanks @AntAISecurityLab

**问题标题**: Hooks/session routing 钩子会话路由
**漏洞描述**: 钩子触发的 `agent:` 会话键未重新绑定到实际目标代理，专用钩子代理丢失会话作用域的工具和插件身份。
**解决方案**:
- 在隔离派发之前将钩子触发的 `agent:` 会话键重新绑定到实际目标代理
- 专用钩子代理保持自己的会话作用域工具和插件身份
- **版本**: 2026.3.31 Thanks @kexinoh

**问题标题**: Discord/voice 语音入口访问控制
**漏洞描述**: 已加入的语音频道在转录前未执行相同的公会频道和成员允许列表检查，配置的 Discord 访问策略之外的用户可提交语音。
**解决方案**:
- 在转录前对语音入口执行相同的公会频道和成员允许列表检查
- 已加入的语音频道不再接受来自配置 Discord 访问策略之外用户的语音
- **版本**: 2026.3.31 Thanks @cyjhhh

**问题标题**: Gateway/tools HTTP 工具调用授权
**漏洞描述**: HTTP 工具调用授权松散，所有者专用工具在 HTTP 调用路径上可用。
**解决方案**:
- 加强 HTTP 工具调用授权
- 所有者专用工具在 HTTP 调用路径上保持禁用
- **版本**: 2026.3.31 (#57773)

## 版本 4.1

### 高危安全案例

**问题标题**: Device pairing 配对授权绕过
**漏洞描述**: 低权限网关调用者可铸造或撤销配对引导材料，内部 `/pair` 设置代码、QR 和清理命令缺乏权限检查。
**解决方案**:
- 内部 `/pair` 设置代码、QR 和清理命令需要 `operator.pairing` 或 `operator.admin`
- 低权限网关调用者不再能铸造或撤销配对引导材料
- **版本**: 2026.4.1 (#60491)

### 中危安全案例

**问题标题**: Chat/error replies 敏感信息泄露
**漏洞描述**: 原始提供者/运行时失败泄露到外部聊天频道，可能暴露内部配置或路径信息。
**解决方案**:
- 停止将原始提供者/运行时失败泄露到外部聊天频道
- 返回友好的重试消息
- 为 Bedrock toolResult/toolUse 会话不匹配添加特定的 `/new` 提示
- **版本**: 2026.4.1 (#58831)

**问题标题**: Agents/failover 提供者错误分类
**漏洞描述**: 提供者特定的 `400`/`422` 载荷被强制转换为通用格式失败，重试、计费或压缩逻辑无法检查它们。
**解决方案**:
- 统一结构化和原始提供者错误分类
- 提供者特定的 `400`/`422` 载荷在重试、计费或压缩逻辑检查之前不再被强制转换为通用格式失败
- **版本**: 2026.4.1 (#58856)

**问题标题**: Auth profiles/store SecretRef 处理
**漏洞描述**: 升级后错误放置的 SecretRef 对象在存储加载期间未从明文 `key` 和 `token` 字段中强制移除，导致代理崩溃。
**解决方案**:
- 在存储加载期间从明文 `key` 和 `token` 字段强制移除错误放置的 SecretRef 对象
- 没有 ACP 运行时的代理在升级后不再因 `.trim()` 崩溃
- **版本**: 2026.4.1 (#58923)

**问题标题**: Auth/OpenAI Codex OAuth 凭证持久化
**漏洞描述**: 插件刷新的 OAuth 凭证在返回前未持久化到 `auth-profiles.json`，轮换的 Codex 刷新令牌在重启后丢失并陷入 `refresh_token_reused` 循环。
**解决方案**:
- 在返回前将插件刷新的 OAuth 凭证持久化到 `auth-profiles.json`
- 轮换的 Codex 刷新令牌在重启后存活
- 不再陷入 `refresh_token_reused` 循环
- **版本**: 2026.4.1 (#53082)

## 版本 4.2

### 高危安全案例

**问题标题**: Sandbox/security 凭证路径绑定绕过
**漏洞描述**: 即使沙箱主路径通过规范别名解析，代理容器仍可通过备用主目录路径挂载用户秘密存储。
**解决方案**:
- 即使沙箱主路径通过规范别名解析，也阻止凭证路径绑定
- 代理容器不能通过备用主目录路径挂载用户秘密存储
- **版本**: 2026.4.2 (#59157)

**问题标题**: Exec/env 环境变量覆盖扩展
**漏洞描述**: 额外的请求作用域环境变量覆盖可重定向包根、语言运行时、编译器包含路径和凭证/配置位置。
**解决方案**:
- 阻止额外的主机环境覆盖枢轴
- 防止重定向包根、语言运行时、编译器包含路径和凭证/配置位置
- 请求作用域执行不能重定向可信工具链或配置查找
- **版本**: 2026.4.2 (#59233)

**问题标题**: Dotenv/workspace overrides 受信任解释器重定向
**漏洞描述**: 工作区 `.env` 文件可覆盖 `OPENCLAW_PINNED_PYTHON` 和 `OPENCLAW_PINNED_WRITE_PYTHON`，可信助手解释器可被仓库本地环境注入重定向。
**解决方案**:
- 阻止工作区 `.env` 文件覆盖 `OPENCLAW_PINNED_PYTHON` 和 `OPENCLAW_PINNED_WRITE_PYTHON`
- 可信助手解释器不能被仓库本地环境注入重定向
- **版本**: 2026.4.2 (#58473)

### 中危安全案例

**问题标题**: Channels/setup 非信任工作区通道插件
**漏洞描述**: 非信任的工作区通道插件在设置解析期间可覆盖内置通道设置/登录流。
**解决方案**:
- 在设置解析期间忽略非信任的工作区通道插件
- 阴影工作区插件不能覆盖内置通道设置/登录流，除非在配置中显式信任
- **版本**: 2026.4.2 (#59158)

**问题标题**: Webhooks/secret comparison 时序攻击
**漏洞描述**: BlueBubbles、飞书、Mattermost、Telegram、Twilio 和 Zalo webhook 处理程序中的时序安全秘密比较不一致，且 BlueBubbles 接受空认证令牌。
**解决方案**:
- 用共享的 `safeEqualSecret` 助手替换临时时序安全秘密比较
- 在 BlueBubbles 中拒绝空认证令牌
- **版本**: 2026.4.2 (#58432)

**问题标题**: OpenShell/mirror 远程工作区约束
**漏洞描述**: `remoteWorkspaceDir` 和 `remoteAgentWorkspaceDir` 未约束到管理的 `/sandbox` 和 `/agent` 根，镜像同步在配置同步期间可覆盖或移除用户添加的 shell 根。
**解决方案**:
- 将 `remoteWorkspaceDir` 和 `remoteAgentWorkspaceDir` 约束到管理的 `/sandbox` 和 `/agent` 根
- 镜像同步不再在配置同步期间覆盖或移除用户添加的 shell 根
- **版本**: 2026.4.2 (#58515)

**问题标题**: Browser/CDP 跨主机枢轴
**漏洞描述**: 初始和发现的 CDP websocket 端点在连接前未验证，严格 SSRF 策略未能阻止跨主机枢轴和直接 websocket 目标。
**解决方案**:
- 在连接前验证初始和发现的 CDP websocket 端点
- 严格 SSRF 策略阻止跨主机枢轴和直接 websocket 目标
- **版本**: 2026.4.2 (#60469)

**问题标题**: Browser/profiles 远程浏览器 SSRF
**漏洞描述**: 远程浏览器配置文件 `cdpUrl` 值在保存配置前未验证是否违反严格 SSRF 策略。
**解决方案**:
- 在保存配置前拒绝违反严格 SSRF 策略的远程浏览器配置文件 `cdpUrl` 值
- 为阻塞端点提供更清晰的验证错误
- **版本**: 2026.4.2 (#60477)

**问题标题**: Gateway/session kill 未授权探测
**漏洞描述**: HTTP 操作员作用域未在会话终止请求上强制执行，未认证调用者可探测会话存在。
**解决方案**:
- 在会话终止请求上强制执行 HTTP 操作员作用域
- 在会话查找之前进行授权检查
- 未认证调用者不能探测会话存在
- **版本**: 2026.4.2 (#59128)

**问题标题**: QQBot/structured payloads 本地文件路径
**漏洞描述**: 本地文件路径未限制到 QQ Bot 拥有的媒体存储，可在该根之外遍历。
**解决方案**:
- 将本地文件路径限制到 QQ Bot 拥有的媒体存储
- 阻止在该根之外遍历
- 减少日志中的路径泄露
- 保持内联图像数据 URL 正常工作
- **版本**: 2026.4.2 (#58453)

## 版本 4.5

### 高危安全案例

**问题标题**: Browser/SSRF 重定向绕过
**漏洞描述**: 浏览器工具在处理交互驱动的导航时未重新执行 SSRF 安全检查，点击、提交、键盘和当前页面快照流在禁止目标上未失败关闭。
**解决方案**:
- 在交互驱动导航后和快照读取前重新运行 SSRF 安全检查
- 点击、提交、键盘和当前页面快照流在禁止目标上失败关闭

**问题标题**: Webhooks/security SecretRef 秘密重新解析
**漏洞描述**: SecretRef 支持的 webhook 路由秘密未在每个请求上重新解析，`openclaw secrets reload` 不能立即撤销之前的秘密。
**解决方案**:
- 在每个请求上重新解析 SecretRef 支持的 webhook 路由秘密
- `openclaw secrets reload` 立即撤销之前的秘密
- 不再等待网关重启
- **版本**: 2026.4.11 (#70727)

**问题标题**: Security/dotenv 端点设置覆盖
**漏洞描述**: 工作区 `.env` 覆盖未阻止 Matrix、Mattermost、IRC 和 Synology 端点设置，克隆的工作区可通过本地端点配置重定向捆绑连接器流量。
**解决方案**:
- 阻止工作区 `.env` 覆盖用于 Matrix、Mattermost、IRC 和 Synology 端点设置
- 克隆的工作区不能通过本地端点配置重定向捆绑连接器流量
- **版本**: 2026.4.11 (#70240)

**问题标题**: Telegram/model picker 未授权访问
**漏洞描述**: 组模型选择器回调未要求相同的 `/models` 授权，未授权参与者可浏览或更改会话模型。
**解决方案**:
- 组模型选择器回调要求相同的 `/models` 授权
- 未授权参与者不能浏览或更改会话模型
- **版本**: 2026.4.11 (#70235)

**问题标题**: Gateway/Control UI 未认证配置读取
**漏洞描述**: 启用 `gateway.auth` 时 `/__openclaw/control-ui-config.json` 未要求认证 Control UI 读取访问，未认证调用者可读取引导元数据。
**解决方案**:
- 启用 `gateway.auth` 时要求认证 Control UI 读取访问用于 `/__openclaw/control-ui-config.json`
- 未认证调用者不能读取引导元数据
- **版本**: 2026.4.11 (#70247)

**问题标题**: Security/external content 特殊令牌清理
**漏洞描述**: 常见的自托管 LLM 聊天模板特殊令牌字面量（包括 Qwen/ChatML、Llama、Gemma、Mistral、Phi 和 GPT-OSS 标记器）未从包装的外部内容和元数据中剥离，存在令牌化器层角色边界欺骗。
**解决方案**:
- 从包装的外部内容和元数据中剥离常见自托管 LLM 聊天模板特殊令牌字面量
- 包括 Qwen/ChatML、Llama、Gemma、Mistral、Phi 和 GPT-OSS 标记器
- 防止令牌化器层角色边界欺骗对保留用户文本中特殊令牌的 OpenAI 兼容后端
- **版本**: 2026.4.11

**问题标题**: Auth/commands 所有者身份要求
**漏洞描述**: 所有者强制命令未要求所有者身份，通配符通道 `allowFrom` 或空所有者候选列表被视为充分，`enforceOwnerForCommands=true` 且 `commands.ownerAllowFrom` 未设置时非所有者发送者可通过宽松回退到达仅所有者命令。
**解决方案**:
- 仅所有者命令要求所有者身份（所有者候选匹配或内部 `operator.admin`）
- 不将通配符通道 `allowFrom` 或空所有者候选列表视为充分
- 非所有者发送者不能通过宽松回退到达仅所有者命令
- **版本**: 2026.4.11 (#69774)

**问题标题**: Synology Chat SSRF via webhook
**漏洞描述**: 出站 webhook `file_url` 值未针对共享 SSRF 策略验证，NAS 可作为困惑代理获取内部地址。
**解决方案**:
- 在转发到 NAS 前验证出站 webhook `file_url` 值针对共享 SSRF 策略
- 拒绝格式错误的 URL、非 `http(s)` 方案和私有/阻塞网络目标
- NAS 不能作为困惑代理获取内部地址
- **版本**: 2026.4.11 (#69784)

**问题标题**: Gateway/Control UI avatar 未认证访问
**漏洞描述**: Control UI avatar 路由（`GET /avatar/<agentId>` 和 `?meta=1` 元数据）在配置认证时未要求网关认证，与同级助手媒体路由不匹配。
**解决方案**:
- 配置认证时要求网关认证用于 Control UI avatar 路由
- 与同级助手媒体路由匹配
- 通过 UI avatar 获取传播现有网关令牌
- **版本**: 2026.4.11 (#69775)

**问题标题**: Google Chat/auth gaxios SSRF 保护
**漏洞描述**: Google 认证 `gaxios` 垫片未使用作用域 SSRF 保护的传输，服务账户认证端点未针对可信 Google URL 验证。
**解决方案**:
- 用作用域 SSRF 保护的传输替换 Google 认证 `gaxios` 垫片
- 针对可信 Google URL 验证服务账户认证端点
- 让插件拥有其分阶段的 `gaxios` 认证运行时
- **版本**: 2026.4.11

**问题标题**: Gateway/MCP loopback 所有者工具可见性
**漏洞描述**: 所有者专用工具可见性从调用者控制的所有者头派生而非区分认证的所有者与非所有者回环承载者，非所有者 MCP 子进程可通过欺骗请求元数据恢复所有者访问。
**解决方案**:
- 从区分认证的所有者与非所有者回环承载者派生仅所有者工具可见性
- 不从调用者控制的所有者头派生
- 非所有者 MCP 子进程不能通过欺骗请求元数据恢复所有者访问
- **版本**: 2026.4.11 (#69796)

## 安全架构改进

### 1. 信任模型明确化

OpenClaw 明确了其"单一用户信任模型"：

- **一个网关 = 一个受信任的操作员**
- 认证的网关调用者被视为该网关实例的受信任操作员
- 会话标识符是路由控制，不是每用户授权边界
- 不建议在多个互不信任的用户之间共享一个网关

### 2. 插件信任边界

- 插件在网关进程中加载，被视为受信任代码
- 插件可以以与 OpenClaw 进程相同的操作系统权限执行
- 只安装您信任的插件，优先使用 `plugins.allow` 固定显式受信任的插件 ID

### 3. 执行沙箱选项

```json
{
  "agents": {
    "defaults": {
      "sandbox": {
        "mode": "non-main" // 或 "all" 以获得更强隔离
      }
    }
  },
  "tools": {
    "exec": {
      "applyPatch": {
        "workspaceOnly": true
      },
      "host": "auto"
    },
    "fs": {
      "workspaceOnly": true
    }
  }
}
```

### 4. 网络边界强化

- **SSRF 防护**: 使用 DNS 固定的调度器
- **代理支持**: 受信任的环境代理模式
- **私有网络**: 显式选择加入私有网络访问

### 5. 临时文件边界

OpenClaw 使用专用的临时根目录：

- 首选: `/tmp/openclaw`
- 回退: `os.tmpdir()/openclaw`
- 沙箱媒体验证仅允许 OpenClaw 管理的临时根下的绝对临时路径

## 安全最佳实践建议

### 1. 部署配置

```bash
# 保持网关仅回环
gateway.bind="loopback"

# 使用强认证
gateway.auth.mode="token" # 或 "password"

# 启用沙箱
agents.defaults.sandbox.mode="non-main"

# 限制文件系统访问
tools.fs.workspaceOnly=true
tools.exec.applyPatch.workspaceOnly=true
```

### 2. 插件管理

```json
{
  "plugins": {
    "allow": [
      "openai",
      "anthropic",
      "slack"
      // 仅列出您信任的插件
    ]
  }
}
```

### 3. 执行审批

```json
{
  "execApprovals": {
    "mode": "allowlist",
    "approvers": ["owner@example.com"],
    "timeoutMinutes": 30
  }
}
```

### 4. 网络安全

- 不要将网关暴露在公共互联网
- 使用 SSH 隧道或 Tailscale 进行远程访问
- 启用 `gateway.auth.mode` 并使用强密钥

### 5. 定期安全审计

```bash
# 运行安全审计
openclaw security audit --deep

# 应用修复
openclaw security audit --fix

# 运行 doctor 检查
openclaw doctor --fix
```
