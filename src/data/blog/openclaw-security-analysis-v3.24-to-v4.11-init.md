本报告分析了 OpenClaw 从版本 3.24 到 4.11 期间的重要安全变更和修复。OpenClaw 是一个 AI 代理框架，其安全模型基于"受信任的操作员"（Trusted Operator）模型，强调单一用户、单一网关的设计理念。

## 版本时间线

- **v3.24 系列**: 早期稳定版本
- **v4.0 系列**: 架构重构版本
- **v4.11 (2026.4.8+)**: 最新稳定版本

## 安全修复分类统计

### 1. SSRF (服务器端请求伪造) 防护

#### MS Teams 文件上传 SSRF 漏洞

**漏洞描述**: Microsoft Teams 的 `fileConsent/invoke` 端点在上传附件时未验证 `uploadInfo.uploadUrl`，攻击者可以通过构造恶意 URL 访问内部网络资源。

**解决方案**:

- 验证上传 URL 必须使用 HTTPS 协议
- 检查主机名是否在 Microsoft/SharePoint 允许列表中
- 执行私有 IP DNS 检查，阻止 SSRF 攻击

**版本**: 2026.4.7 (#23596)

#### QQ Bot 图片尺寸探测 SSRF

**漏洞描述**: QQ Bot 在探测图片尺寸时可能被利用访问内部资源。

**解决方案**:

- 通过受保护的 QQ/Tencent HTTPS 获取附件
- 不再跟随任意远程主机

**版本**: 2026.4.7 (#63495)

#### 浏览器 SSRF 重定向绕过

**漏洞描述**: 浏览器工具在处理交互驱动的导航时未重新执行 SSRF 安全检查。

**解决方案**:

- 在交互驱动导航后和快照读取前重新运行 SSRF 安全检查
- 阻止不允许的目标访问

**版本**: 2026.4.5 (#62023)

#### 网络获取防护重定向信息泄露

**漏洞描述**: 跨源 307/308 重定向时未丢弃请求体和描述性头部，可能导致敏感信息泄露。

**解决方案**:

- 默认丢弃请求体和 body-describing 头部
- 防止攻击者控制的重定向接收包含秘密的 POST 载荷

**版本**: 2026.4.7 (#62357)

### 2. 环境变量注入防护

#### 主机执行环境变量注入

**漏洞描述**: 主机执行工具未过滤危险的环境变量，攻击者可以通过设置特定环境变量重定向代码执行、配置路径或凭证。

**解决方案**:

- 阻止 Java、Rust、Cargo、Git、Kubernetes、云凭证、配置路径、Helm 等危险环境变量覆盖
- 防止主机运行的工具被重定向到攻击者选择的代码、配置、凭证或仓库状态

**版本**: 2026.4.7 (#59119, #62002, #62291)

**具体阻止的环境变量**:

```bash
# Git 相关
GIT_DIR, GIT_WORK_TREE, GIT_COMMON_DIR, GIT_INDEX_FILE
GIT_OBJECT_DIRECTORY, GIT_ALTERNATE_OBJECT_DIRECTORIES, GIT_NAMESPACE

# 云服务凭证
KUBECONFIG, AWS_*, AZURE_*, GCP_*

# 包管理器
CARGO_HOME, HELM_HOME, NPM_CONFIG_*

# Python 包索引
PIP_INDEX_URL, PIP_EXTRA_INDEX_URL
```

### 3. 认证与授权强化

#### 网关认证令牌轮换后会话未失效

**漏洞描述**: 当共享令牌或密码轮换时，现有的 WebSocket 会话仍然保持连接，可能导致旧凭证继续有效。

**解决方案**:

- 令牌或密码更改时使现有 WebSocket 会话失效
- 防止过期的认证会话继续访问

**版本**: 2026.4.7 (#62350)

#### 设备配对令牌作用域升级

**漏洞描述**: 设备配对令牌可以被轮换到配对时未批准的角色。

**解决方案**:

- 拒绝将设备令牌轮换到配对期间从未批准的角色
- 保持重新连接角色检查绑定到配对设备的批准角色集

**版本**: 2026.4.5 (#60462)

#### 允许列表命令授权绕过

**漏洞描述**: 非所有者但具有命令授权的发送者可以持久重写允许列表策略状态。

**解决方案**:

- `/allowlist add` 和 `/allowlist remove` 需要所有者授权
- 在通道解析前验证所有者权限

**版本**: 2026.4.7 (#62383)

#### 网关插件路由权限提升

**漏洞描述**: 插件 HTTP 处理程序在缺少或不受信任的 HTTP scope 头部时可能获得管理员级运行时作用域。

**解决方案**:

- 保持网关认证插件运行时路由在只写回退作用域上
- 除非受信任代理调用者显式声明更窄的 `x-openclaw-scopes`

**版本**: 2026.4.5 (#59815)

### 4. 路径遍历与文件系统安全

#### 媒体服务器路径遍历漏洞

**漏洞描述**: 媒体服务器未正确验证文件路径，可能允许访问预期目录之外的文件。

**解决方案**:

- 实施严格的路径验证
- 阻止路径遍历攻击

**Git Commit**: b94b220156

#### 飞书文档上传工作区限制绕过

**漏洞描述**: 当启用 `tools.fs.workspaceOnly` 时，飞书文档上传仍可读取工作区外的本地文件。

**解决方案**:

- 将工作区约束的 `localRoots` 转发到媒体加载器
- 阻止工作区外文件读取

**版本**: 2026.4.7 (#62369)

#### 沙箱文件系统桥接写入路径锚定

**漏洞描述**: 沙箱文件系统桥接的写入操作未正确锚定到规范父路径。

**解决方案**:

- 将 writeFile 提交锚定到规范父路径
- 防止路径逃逸

**版本**: 2026.4.5 (#5335)

### 5. 命令注入防护

#### iMessage SCP 远程路径 shell 元字符注入

**漏洞描述**: iMessage 的 SCP 远程路径未正确清理，可能允许 shell 元字符注入。

**解决方案**:

- 清理 SCP 远程路径
- 防止 shell 元字符注入

**Git Commit**: a54bf71b4c

#### 未引用 heredoc 主体中的命令替换

**漏洞描述**: 未引用的 heredoc 主体可能执行命令替换。

**解决方案**:

- 阻止未引用 heredoc 主体中的命令替换

**Git Commit**: 92cada2aca, a54bba6ee0

#### 飞书正则表达式元字符注入

**漏洞描述**: 飞书在清理机器人提及时未转义正则表达式元字符。

**解决方案**:

- 在 stripBotMention 中转义正则表达式元字符

**Git Commit**: 7e67ab75cc

### 6. 执行审批与沙箱安全

#### 浏览器配置变更绕过执行审批

**漏洞描述**: 模型可以通过 `gateway config.apply` 和 `config.patch` 修改执行审批路径配置。

**解决方案**:

- 阻止模型面向的配置写入更改执行审批路径
- 包括 `safeBins`, `safeBinProfiles`, `safeBinTrustedDirs`, `strictInlineEval`

**版本**: 2026.4.7 (#62001)

#### 节点执行审批操作数绑定漂移

**漏洞描述**: 节点主机 `system.run` 审批在异步转发期间可能丢失操作数绑定。

**解决方案**:

- 保持节点主机审批绑定到准备的执行计划
- 可变脚本操作数仍获得审批时绑定和漂移重新验证

**版本**: 2026.4.7 (#62401, #62439)

#### 沙箱远程文件系统桥接读取竞争条件

**漏洞描述**: 沙箱远程文件系统桥接读取存在路径解析竞争条件。

**解决方案**:

- 使远程文件系统桥接读取锚定父路径并原子打开文件
- 防止读取访问竞争路径解析

**版本**: 2026.4.5 (#58024)

### 7. 插件与扩展安全

#### ClawHub 插件归档完整性验证

**漏洞描述**: 从 ClawHub 安装插件时未验证归档完整性。

**解决方案**:

- 验证下载的插件归档与版本元数据 SHA-256
- 归档完整性元数据缺失或格式错误时失败关闭
- 加强国退 ZIP 验证

**版本**: 2026.4.7 (#60517)

#### 插件市场符号链接逃逸

**漏洞描述**: 远程市场安装可能通过符号链接逃逸。

**解决方案**:

- 阻止远程市场符号链接逃逸
- 保持普通本地市场安装路径

**版本**: 2026.4.5 (#60556)

#### Lobster 任意执行漏洞

**漏洞描述**: 通过 `lobsterPath/cwd` 可能执行任意代码。

**解决方案**:

- 阻止通过 lobsterPath/cwd 的任意执行

**GHSA**: GHSA-4mhr-g7xj-cg8j

**版本**: 2026.4.5 (#5335)

### 8. 浏览器与 Canvas 安全

#### 浏览器配置文件持久化变更绕过

**漏洞描述**: 通过 `browser.proxy` 可以在网关转发和节点主机代理路径上创建、重置和删除浏览器配置文件。

**解决方案**:

- 阻止通过 `browser.proxy` 的持久浏览器配置文件变更
- 即使未配置配置文件允许列表也生效

**版本**: 2026.4.7 (#60489)

#### Android Canvas 安全端点处理

**漏洞描述**: Android canvas 桥接操作未严格验证 URL。

**解决方案**:

- 要求精确规范化的 A2UI URL 匹配
- 拒绝查询不匹配和后代路径
- 仍允许仅片段的 A2UI 导航

**版本**: 2026.4.5 (#60128, #60208, #60221)

### 9. 内存与上下文安全

#### Base64 解码字节限制绕过

**漏洞描述**: Teams、Signal、QQ Bot 和图像工具的 base64 载荷在解码前未强制执行字节限制。

**解决方案**:

- 在解码错过的 base64 载荷前强制执行字节限制
- 防止超大入站媒体和数据 URL 绕过预解码大小检查

**版本**: 2026.4.7 (#62007)

#### 运行时事件信任边界

**漏洞描述**: 后台 `notifyOnExit` 摘要、ACP 父流中继和唤醒钩子载荷被标记为受信任的系统事件。

**解决方案**:

- 将这些事件标记为不受信任的系统事件
- 防止低信任运行时输出作为受信任的 `System:` 文本重新进入后续轮次

**版本**: 2026.4.7 (#62003)

### 10. Claude CLI 后端安全

#### Claude CLI 配置根环境变量覆盖

**漏洞描述**: OpenClaw 启动的 Claude CLI 运行可能被指向具有不同钩子、插件或认证上下文的备用配置/插件树。

**解决方案**:

- 清除继承的 Claude Code 配置根和插件根环境变量覆盖
- 包括 `CLAUDE_CONFIG_DIR` 和 `CLAUDE_CODE_PLUGIN_*`

**版本**: 2026.4.5

#### Claude CLI 提供者路由环境变量覆盖

**漏洞描述**: Claude CLI 后门会话可能被静默重定向到代理、Bedrock、Vertex、Foundry 或父管理的令牌上下文。

**解决方案**:

- 清除继承的 Claude Code 提供者路由和托管认证环境变量覆盖
- 将 OpenClaw 启动的 Claude CLI 运行标记为主机管理

**版本**: 2026.4.5

#### Claude CLI 权限模式绕过

**漏洞描述**: 自定义 `cliBackends.claude-cli.args` 可能意外使用错误的权限模式。

**解决方案**:

- 强制主机管理的 Claude CLI 运行使用 `--setting-sources user`
- 防止仓库本地 `.claude` 设置在非交互会话中执行
- 将格式错误的 `--permission-mode` 视为缺失并安全回退

**版本**: 2026.4.5

## 安全架构改进

### 信任模型明确化

OpenClaw 明确了其"单一用户信任模型"：

- **一个网关 = 一个受信任的操作员**
- 认证的网关调用者被视为该网关实例的受信任操作员
- 会话标识符是路由控制，不是每用户授权边界
- 不建议在多个互不信任的用户之间共享一个网关

### 插件信任边界

- 插件在网关进程中加载，被视为受信任代码
- 插件可以以与 OpenClaw 进程相同的操作系统权限执行
- 只安装您信任的插件，优先使用 `plugins.allow` 固定显式受信任的插件 ID

### 执行沙箱选项

```json
{
  "agents": {
    "defaults": {
      "sandbox": {
        "mode": "non-main"
      }
    }
  },
  "tools": {
    "exec": {
      "applyPatch": {
        "workspaceOnly": true
      },
      "host": "auto"
    },
    "fs": {
      "workspaceOnly": true
    }
  }
}
```

### 网络边界强化

- **SSRF 防护**: 使用 DNS 固定的调度器
- **代理支持**: 受信任的环境代理模式
- **私有网络**: 显式选择加入私有网络访问

### 临时文件边界

OpenClaw 使用专用的临时根目录：

- 首选: `/tmp/openclaw`
- 回退: `os.tmpdir()/openclaw`
- 沙箱媒体验证仅允许 OpenClaw 管理的临时根下的绝对临时路径

## 安全最佳实践建议

### 部署配置

```bash
# 保持网关仅回环
gateway.bind="loopback"

# 使用强认证
gateway.auth.mode="token" # 或 "password"

# 启用沙箱
agents.defaults.sandbox.mode="non-main"

# 限制文件系统访问
tools.fs.workspaceOnly=true
tools.exec.applyPatch.workspaceOnly=true
```

### 插件管理

```json
{
  "plugins": {
    "allow": [
      "openai",
      "anthropic",
      "slack"
    ]
  }
}
```

仅列出您信任的插件。

### 执行审批

```json
{
  "execApprovals": {
    "mode": "allowlist",
    "approvers": ["owner@example.com"],
    "timeoutMinutes": 30
  }
}
```

### 网络安全

- 不要将网关暴露在公共互联网
- 使用 SSH 隧道或 Tailscale 进行远程访问
- 启用 `gateway.auth.mode` 并使用强密钥

### 定期安全审计

```bash
# 运行安全审计
openclaw security audit --deep

# 应用修复
openclaw security audit --fix

# 运行 doctor 检查
openclaw doctor --fix
```

## Node.js 版本要求

OpenClaw 要求 **Node.js 22.12.0 或更高版本**，包含重要安全补丁：

- **CVE-2025-59466**: async_hooks DoS 漏洞
- **CVE-2026-21636**: 权限模型绕过漏洞

```bash
# 验证 Node.js 版本
node --version  # 应为 v22.12.0 或更高
```

## Docker 安全建议

```bash
docker run --read-only --cap-drop=ALL \
  -v openclaw-data:/app/data \
  openclaw/openclaw:latest
```

- 官方镜像以非 root 用户 (`node`) 运行
- 使用 `--read-only` 进行额外的文件系统保护
- 使用 `--cap-drop=ALL` 限制容器能力

## 总结

OpenClaw 从 v3.24 到 v4.11 版本期间进行了大量的安全强化工作，主要集中在：

1. **SSRF 防护**: 全面的服务器端请求伪造防护
2. **注入防护**: 环境变量、命令、路径遍历等多种注入攻击防护
3. **认证强化**: 令牌管理、会话失效、权限边界强化
4. **执行安全**: 沙箱隔离、审批流程、操作数绑定
5. **插件安全**: 完整性验证、信任边界明确化

建议所有用户：

- 升级到最新版本 (v4.11+)
- 使用 Node.js 22.12.0+
- 遵循最小权限原则配置
- 定期运行安全审计
- 保持插件列表最小化和受信任
